# 播客音频或视频剪辑 Skill（中文版）

> 这是 [SKILL.md](SKILL.md) 的中文镜像，便于阅读和协作。Codex 实际加载入口仍为英文 `SKILL.md`；两份文件发生差异时，以 `SKILL.md` 为准。

保留原始媒体不变。使用可编辑的 Markdown 作为控制层；用户最新保存的文档高于此前的提案和聊天总结。

## 运行依赖与可调用工具

开始时检查以下项目，并且只使用所选路线实际需要的工具：

- 必需媒体工具：`ffmpeg`、`ffprobe`，以及 `exec_command` 等可调用的 shell。
- 必需文件操作：文件搜索与读取，以及 `apply_patch` 或等效的安全写入调用。
- 必需转写路线：本地任务优先使用内置 `faster-whisper` 包装脚本，也可以使用其他已安装的语音转文字引擎、可用的转写函数/API，或用户提供的逐字稿。
- 可选检查工具：`view_image`、帧提取、波形/频谱工具和响度分析。
- 可选语言路线：使用本地模型或可用翻译函数生成多语言字幕；始终标明机器翻译需要人工复核。
- 可选视觉路线：已经授权的 Google Photos/Drive 连接器、授权清晰的媒体搜索、HTML/canvas 工作流、`image_gen.imagegen`，或可用的视频生成函数。通过当前工具注册表发现连接器和生成函数；仅在 `full-visual` 模式且获得授权后使用。唯一例外是已要求的自媒体封面：任何原素材类型都可使用已经授权的图像生成或设计路线制作。
- 可选精修：需要人工精修时，使用原生 NLE，并可交付 EDL/XML/FCPXML/Resolve 项目。
- 缺少软件或连接器时：在安装、下载、调用连接器或生成媒体之前，使用当前环境的审批/提权调用或插件安装请求。

记录实际使用的工具，并区分自动检查与实际观看/试听。

## 内置可复用资源

- 从 `assets/templates/` 复制空白控制文档，不要重新创建其结构。
- 只有需要本地/离线转写或字幕翻译时，才读取 [references/local-models.md](references/local-models.md)。
- 本地转写使用 `scripts/transcribe_faster_whisper.py`。确定性字幕处理分别使用 `scripts/srt_merge_cues.py`、`scripts/srt_translate_ollama.py`、`scripts/srt_bilingual.py`、`scripts/srt_remap_timeline.py` 和 `scripts/srt_to_markdown.py`。先运行 `--help`，并保护用户已经修改的字幕文件。

## 根据原素材选择路线

- `视频 + audio-led`：只根据对白和声音决定剪辑。每个剪切点都对音频和视频使用完全相同的边界，不增加独立画面剪辑。
- `视频 + full-visual`：允许加入已经批准的多机位、重构图、图形或 B-roll，并记录来源和权利信息。
- `纯音频`：剪辑和母带处理播客音频。跳过视频规划、视频导出、YouTube 文案和小红书文案。唯一允许的视觉交付物是用户要求的自媒体封面；只有用户要求发布资料时才生成播客 feed 元数据。

确定性剪辑优先使用 FFmpeg；复杂人工视觉精修优先使用原生 NLE。只有媒体已经被解码并检查过，才能声称实际观看或听取过。

## 阶段 0：快速预检和简报

1. 读取适用的指南：中文任务使用 [references/editing-guide.zh-CN.md](references/editing-guide.zh-CN.md)，英文任务使用 [references/editing-guide.md](references/editing-guide.md)。
2. 只盘点相关的原素材、参考节目、音乐和交付文件。记录编码、时长；视频还要记录分辨率/帧率；音频记录采样率/声道和可测量响度。
3. 根据 `assets/templates/episode-brief.md` 创建或刷新 `output/episode-brief.md`。只要求核心目的、原素材类型、处理路线、排除项、目标平台/时长、参考材料、字幕语言/格式、封面要求和其他不可妥协要求。不要要求 intended audience effect、desired tone 或 must-keep material。
4. 只有某项信息会阻塞内容判断或权利决定时才向用户提问。安全的技术默认值应直接推断并写入记录，不增加审批步骤。

## 阶段 1：第一轮内容方案

1. 生成包含时间码和说话人标签的逐字稿；不确定的词需要标记，不能猜测。
2. 只有用户提供参考节目时才进行分析，并且只分析用户指定的属性。
3. 识别故事结构、有力回答、重复内容、口误重来、长停顿、跑题段落、敏感内容、技术问题和短视频候选片段。
4. 分析完整录音后提出冷开场精彩混剪。使用完整陈述，并记录原始时间码、拼接顺序、时长和语境风险。不得通过拼接残句制造新的主张。
5. 单独编写冷开场音乐简报。先检查用户放在 `assets/` 的候选音乐；否则提出使用授权清晰/用户提供的音乐，或经过授权的生成路线。即使复用同一首音乐，也要分别决定冷开场、正式片头、章节和片尾音乐。
6. 根据内置模板生成 `output/round-1-content-plan.md`、`output/transcript.md` 和 `output/subtitle-plan.md`。只有存在参考节目时才创建 `output/reference-style.md`。
7. 用户要求发布资料或封面时，根据 `assets/templates/cover-brief.md` 创建 `output/cover-brief.md`。先检查 `assets/` 中相关视觉参考（如过往封面、Logo、字体、色板、版式和品牌指南），并将其作为首要风格参考；没有参考时，再从节目内容中推导干净、符合平台需求的方向，并明确记录。不得复制第三方作品、使用未授权的人像或 Logo，也不得在封面上写入没有依据的主张。

## 阶段 2：第二轮时间线方案

1. 重新读取最新的节目简报和第一轮方案。
2. `纯音频`：映射音频剪切、音乐、字幕/逐字稿和导出；删除所有视觉字段。
3. `视频 + audio-led`：把已批准的音频剪切映射为完全相同的视频剪切，并明确不允许独立画面剪辑。
4. `视频 + full-visual`：记录每个插入素材的原始时间、时间线位置、时长、呈现位置、进入/退出方式、音频行为、目的、来源、权利和审批状态。
5. 生成 `output/round-2-timeline-plan.md`、`output/music-plan.md` 和 `output/asset-log.md`；只写实际使用素材所需的细节。

简单项目或 audio-led 项目可以一次生成两轮文档，并只设置一个合并审批点。只有涉及实质性删改、敏感段落、外部素材、新音乐或生成选择时才单独暂停等待决定。

## 阶段 3：渲染与修改

1. 只执行已经批准的实质性修改。删除明显口误重来和重复录制，同时保留自然呼吸、笑声、情绪和对话节奏。
2. 保守修复对白，避免金属感降噪伪影和过度压缩。
3. 谨慎使用音乐，并在人声下方自动避让。发布前确认用户放在 `assets/` 中的音乐权利。
4. 生成用户要求的每一种字幕格式和语言。只有用户明确要求时才烧录字幕。
5. 为每个已要求的平台/宽高比制作获批的自媒体封面。若 `assets/` 中有视觉参考，应保持其风格；否则遵循已确认的封面简报。导出清晰易读的 PNG 或 JPG；所选工具支持时保留可编辑源文件。未经授权，不得使用真实人物的生成肖像。
6. 对复杂或不确定的修改先生成低成本预览。方案已经确定且明确批准时，直接生成交付版并质检，以节省时间。
7. 把修改记录在 `output/edit-log-v1.md`，并保留此前版本。

## 阶段 4：审核、母带和打包

1. 连续检查实际媒体。视频任务检查音画同步；所有任务检查截断词、突兀剪切、静音、音频伪影、峰值、字幕时间、拼写、翻译、音乐转场/权利和结尾留白。
2. 创建 `output/ai-final-review.md`，区分客观缺陷和主观建议。未经批准，不执行新的主观修改。
3. 根据已批准路线导出：
   - 视频原素材：最终视频、WAV 和压缩播客音频；
   - 纯音频原素材：WAV 母带和 MP3/M4A，不生成视频交付物。
4. 没有现成制作规范时，默认播客响度约为立体声 -16 LUFS 或单声道 -19 LUFS，真峰值不高于 -1 dBTP，并进行实际测量。
5. 只为用户要求的平台生成发布资料。纯音频输入生成播客 feed 标题、摘要、章节、标签/分类、版权信息和 manifest；绝不生成 YouTube 或小红书视频素材。任何原素材类型均可交付已要求的自媒体封面。
6. 以最终像素尺寸检查每个交付封面：确认目标平台/宽高比、缩略图尺寸下的标题和人物可读性、安全边距、人名和主张准确、参考风格一致，以及图片/素材权利。将封面路径、尺寸、参考素材和生成/设计方式记录在 `output/qc-report.md` 与 `output/asset-log.md`。
7. 验证媒体/字幕路径、JSON、章节、人名和事实性主张，创建 `output/qc-report.md`。

## 内容安全边界

- 不得通过删除或重排片段改变说话者原意。
- 法律、保密、争议性或可能影响声誉的内容必须明确标记并获得批准。
- 不得伪造反应镜头、事件、产品画面、引语或证据。
- 合成说明性媒体可能被误认为纪实证据时，必须明确标注。
- 发布前必须由人工完整观看或试听。
